> 本系列页面基于 `/apifox/openai-openapi.yaml` 生成，涵盖 Chat、Responses、音视频、图像等全部兼容接口。

kapon 为 OpenAI 生态提供完全兼容的代理层：

- **同样的路径与参数**：保持与官方 REST API 一致，可直接复用现有 SDK/代码。
- **统一认证**：使用 kapon 控制台签发的 Token 即可访问多个渠道，支持细粒度的密钥权限与监控。
- **多云路由**：同一套调用可在 kapon 控制台中按需切换后端供应商或冗余策略。

<Note>
  填写 `Authorization: Bearer <kapon Token>` 作为认证头；也可在 Mintlify Playground 中统一配置。
</Note>

```bash
export BASE_URL="https://models.kapon.cloud"
export TOKEN="oh-xxxxxxxxxxxxxxxx"
```

配置好环境变量后，可直接用 cURL 调试：

```bash
curl "$BASE_URL/v1/models" \
  -H "Authorization: Bearer $TOKEN"
```

## 使用建议

- 推荐将 Base URL 抽象为环境变量或配置项，方便在多个环境间切换。
- 若需要流式响应，请在 cURL 中加入 `-N` 并处理分块；Python `requests` 可设置 `stream=True`。
- 对于大文件上传（语音转写、图片编辑），优先使用 SDK 的流式上传能力以避免内存溢出。

更多接口详情请通过左侧导航进入对应页面，或直接展开自动生成的 OpenAPI 参考。
